## 프로젝트 개요

### 서비스 소개
**쓱싹(SsgSak)**은 홈케어 서비스(설치/시공, 클리닝, 막힘해결)를 제공하는 전문가-고객 매칭 플랫폼입니다.

### 비즈니스 목표
- 전문가와 고객을 효율적으로 연결
- 투명한 가격 책정 및 결제 시스템
- 신뢰할 수 있는 서비스 품질 관리
- 공정한 전문가 정산 시스템

### 핵심 사용자
1. **전문가**: 서비스를 제공하는 기술자/업체
2. **운영팀**: 플랫폼을 관리하는 백오피스 사용자
3. **고객**: 서비스를 이용하는 일반 사용자 (향후 개발)

---

## 변경사항 요약

### 기존 계획 (v1.0)
```
백엔드: Supabase (PostgreSQL + Auth + Realtime)
프론트엔드: Next.js (3개 앱)
배포: Vercel
```

### 신규 계획 (v2.0)
```
백엔드: PostgreSQL + Node.js (Express) + Socket.io
프론트엔드: React (전문가 웹앱, 백오피스)
배포: 로컬 개발 → 클라우드 (AWS/GCP/Azure)
```

### 주요 변경 이유
1. **백엔드 독립성**: Supabase 종속성 제거, 자체 API 서버 구축
2. **비용 최적화**: 오픈소스 기반 스택으로 비용 절감
3. **확장성**: 향후 기능 추가 및 커스터마이징 용이
4. **개발 우선순위**: 전문가 웹앱 → 백오피스 → 사용자 웹 순서

## 개발 우선순위

### Phase 1: 전문가 웹앱 (최우선)
**목표**: 전문가가 주문을 관리하고 서비스를 제공할 수 있는 웹 애플리케이션

**핵심 기능**:
- 전문가 회원가입/로그인
- 주문 조회 및 수락/거절
- 서비스 진행 상태 관리
- 일정 관리
- 정산 조회
- 프로필 관리

**예상 기간**: 2-3주

---

### Phase 2: 백오피스 (2순위)
**목표**: 운영팀이 플랫폼을 관리할 수 있는 관리자 시스템

**핵심 기능**:
- 관리자 로그인
- 사용자 관리 (전문가, 고객)
- 주문 관리 및 모니터링
- 서비스 카탈로그 관리
- 정산 처리
- 통계 및 리포트

**예상 기간**: 2-3주

---

### Phase 3: 사용자 웹 (향후)
**목표**: 고객이 서비스를 검색하고 주문할 수 있는 웹 애플리케이션

**핵심 기능**:
- 고객 회원가입/로그인
- 서비스 검색 및 필터링
- 주문 생성 및 결제
- 주문 조회 및 취소
- 리뷰 작성
- 프로필 관리

**예상 기간**: 3-4주

## 기술 스택

### 백엔드
```yaml
언어: Node.js (v18+)
프레임워크: Express.js
데이터베이스: PostgreSQL 14+
캐시: Redis
인증: JWT (jsonwebtoken)
실시간: Socket.io
파일 업로드: Multer
ORM: Prisma 또는 TypeORM
```

### 프론트엔드
```yaml
라이브러리: React 18
언어: TypeScript
상태관리: Zustand 또는 Redux Toolkit
라우팅: React Router v6
UI 프레임워크: Tailwind CSS
HTTP 클라이언트: Axios
폼 관리: React Hook Form
테이블: TanStack Table
```

### DevOps
```yaml
컨테이너: Docker
오케스트레이션: Docker Compose
웹서버: Nginx
CI/CD: GitHub Actions
모니터링: PM2, Winston (로깅)
```

### 외부 서비스
```yaml
결제: Toss Payments
SMS: Aligo 또는 Coolsms
이메일: SendGrid 또는 AWS SES
파일 저장: AWS S3 또는 로컬 스토리지
```

### 데이터베이스 구조
```
users (사용자 기본 정보)
├── customers (고객 상세)
├── experts (전문가 상세)
└── admins (관리자 상세)

service_categories (서비스 대분류)
└── service_items (서비스 세부 항목)

orders (주문)
├── order_notes (주문 메모)
├── order_attachments (첨부파일)
└── service_schedule (일정)

payments (결제)
└── payment_transactions (결제 거래)

settlements (정산)
└── settlement_details (정산 상세)

reviews (리뷰)
notifications (알림)
audit_logs (감사 로그)
```
