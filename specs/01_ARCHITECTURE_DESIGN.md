## 아키텍처 원칙

### 핵심 원칙
1. **관심사 분리**: 백엔드 API, 프론트엔드 앱, 데이터 계층 명확히 분리
2. **모듈성**: 기능별 모듈화로 독립적 개발 및 테스트 가능
3. **타입 안정성**: TypeScript + Prisma로 엔드투엔드 타입 안정성 보장
4. **실시간 통신**: Socket.io 기반 실시간 업데이트 지원
5. **모바일 퍼스트**: 전문가 웹앱은 모바일 환경 최적화

### 설계 패턴
- **백엔드**: 플러그인 기반 Fastify 아키텍처
- **프론트엔드**: 컴포넌트 기반 React + Ant Design
- **상태 관리**: Zustand (간결한 Flux 패턴)
- **데이터 접근**: Prisma Client (Repository 패턴 유사)

### 개발 원칙
- **테스트 주도**: 단위 테스트, 통합 테스트, E2E 테스트 계층화
- **문서 주도**: OpenAPI 스펙 기반 API 문서 자동화
- **코드 리뷰**: 풀 리퀘스트 기반 협업 프로세스

### 컴포넌트 상호작용
1. **클라이언트 → API 서버**: HTTPS 통신, JWT 인증
2. **API 서버 → 데이터베이스**: Prisma Client via connection pool
3. **API 서버 → 외부 서비스**: Axios HTTP 클라이언트
4. **실시간 통신**: Socket.io 웹소켓 연결
5. **파일 업로드**: Multipart form data → S3/로컬 저장

#### 플러그인 아키텍처
Fastify의 플러그인 시스템을 활용한 모듈화:
1. **인증 플러그인**: JWT 검증, 사용자 추출 데코레이터
2. **CORS 플러그인**: 크로스 오리진 리소스 공유 설정
3. **Swagger 플러그인**: OpenAPI 3.0 문서 자동 생성
4. **Socket.io 플러그인**: 실시간 통신 설정
5. **Rate Limit 플러그인**: API 요청 제한

#### 라우트 모듈화
각 도메인별 독립적인 라우트 모듈:
- **의존성 주입**: 서비스 계층을 라우트에 주입
- **스키마 검증**: @sinclair/typebox 기반 런타임 유효성 검증
- **에러 처리**: 통일된 에러 응답 형식
- **OpenAPI 문서**: 라우트 정의 시 문서 자동 생성

### 전문가 관리 시스템 아키텍처

전문가 관리 시스템은 관리자 백오피스와 전문가 웹앱을 통해 전문가의 전체 라이프사이클을 관리하는 통합 모듈입니다. 12가지 상세 시나리오(마스터/서브 계정 등록, 승인, 정보 수정, 활성/비활성 전환, 서비스 카테고리/권역 변경, 멤버십 구좌 설정, 배정 이력 조회, 패널티 적용, 일일 배정 상한 설정, 통계 대시보드)를 지원합니다.

#### 핵심 구성 요소

1. **계정 관리 모듈**
   - 마스터 계정/서브 계정 분리 구조
   - 승인 워크플로우 (PENDING → APPROVED/REJECTED)
   - 활성/비활성 상태 관리

2. **멤버십 관리 모듈**
   - 멤버십 구좌 설정 (membership_slot_count): 마스터 계정 단위로 관리되는 멤버십 구좌 수
   - 가중치 기반 우선 배정 로직: 멤버십 전문가는 membership_slot_count 값만큼 가중치 부여 (일반 전문가 = 1)
   - 멤버십 적용 중분류 제한: 특정 서비스 중분류에만 멤버십 구좌 적용 가능
   - v2/v3 확장 포인트: 일일 배정 상한, 품질 지표 연동, 슬롯 상한 설계, 포인터 스코프 확장

3. **패널티 관리 모듈**
   - SOFT_LIMIT, HARD_BLOCK, CATEGORY_LIMIT, REGION_LIMIT, MEMBERSHIP_SUSPEND 유형
   - 기간 기반 자동 만료 처리
   - 배정 로직 통합 제어

4. **배정 정책 모듈**
   - 일일 배정 상한 (daily_assignment_limit)
   - 실시간 배정 건수 추적
   - 상한 초과 시 자동 배정 제외

5. **통계 및 분석 모듈**
   - 배정 이력 스냅샷 기반 통계
   - 멤버십 효과 분석 지표
   - 패널티 적용 근거 데이터

#### 아키텍처 통합 포인트

- **배정 로직**: 전문가 상태, 멤버십 가중치, 패널티, 일일 상한이 배정 후보군 필터링에 통합
- **알림 시스템**: 승인, 패널티, 멤버십 변경 시 실시간 알림 발송
- **감사 로그**: 모든 관리 작업의 스냅샷 보관 및 변경 이력 추적
- **실시간 동기화**: 전문가 웹앱과 백오피스 간 실시간 상태 동기화 (Socket.io)

#### 데이터 모델 확장

기존 데이터베이스 스키마에 다음 확장이 추가되었습니다:
- `master_account`, `sub_account` 계정 분리
- `master_membership`, `membership_mid_list` 멤버십 관리
- `penalty_history` 패널티 이력
- `master_assignment_policy` 배정 정책
- `assignment_history` 배정 이력 스냅샷
- `audit_logs` 감사 로그

#### 보안 고려사항

- **RBAC 계층화**: 관리자(읽기/쓰기), 전문가(자기 데이터 읽기 전용)
- **데이터 접근 제어**: 전문가는 본인의 정보만 조회 가능 (마스터 계정 기준)
- **감사 추적**: 모든 관리 작업은 관리자 ID 및 시각 기록
- **스냅샷 보존**: 변경 전 데이터 스냅샷으로 무결성 보장

#### 확장성 설계

- **수평 확장**: 전문가 관리 모듈은 독립적으로 확장 가능
- **캐싱 전략**: 자주 조회되는 전문가 프로필은 Redis 캐싱
- **비동기 처리**: 대량 배정 이력 조회는 배치 처리
- **모니터링**: 전문가별 배정 성과 지표 실시간 모니터링
- **참고 문서**: 배정 로직 상세 사용자 시나리오는 `/docs/menu10_assignment_logic_spec_ko.md` 참조

## 데이터 흐름

### 사용자 인증 흐름
```
1. 클라이언트 → 로그인 요청 (email/password)
2. Fastify API → 자격 증명 검증
3. Prisma → 사용자 조회 및 비밀번호 확인
4. Fastify API → JWT 토큰 생성
5. 클라이언트 ← 토큰 응답 (HTTP-only 쿠키 또는 localStorage)
6. 이후 요청 → Authorization 헤더에 토큰 포함
7. Fastify 인증 플러그인 → 토큰 검증 및 사용자 추출
8. 요청 처리 → 권한 검사 (RBAC)
```

### 주문 생성 흐름
```
1. 전문가/고객 → 주문 생성 요청 (서비스, 날짜, 주소 등)
2. Fastify API → 요청 검증 (TypeBox)
3. Order Service → 가격 스냅샷 생성 (현재 서비스 가격 저장)
4. Payment Service → 결제 요청 (Hecto PG)
5. Hecto PG → 결제 처리 및 응답
6. Order Service → 주문 데이터베이스 저장 (Prisma)
7. Notification Service → 알림 전송 (Aligo API - SMS/카카오톡)
8. Socket.io → 실시간 업데이트 브로드캐스트
9. 클라이언트 ← 주문 생성 응답
```

### 실시간 알림 흐름
```
1. 이벤트 발생 (주문 생성, 상태 변경, 결제 완료 등)
2. Notification Service → 알림 생성 (데이터베이스 저장)
3. Socket.io → 연결된 클라이언트에 이벤트 브로드캐스트
4. 동시에 Aligo API → SMS/카카오톡 메시지 전송
5. 클라이언트 (웹소켓) → 실시간 알림 수신 및 UI 업데이트
6. 클라이언트 (앱) → 알림 목록 갱신
```

### 파일 업로드 흐름
```
1. 클라이언트 → multipart/form-data 요청 (파일 포함)
2. Fastify Multipart 플러그인 → 파일 파싱
3. File Service → 파일 유효성 검사 (크기, 형식)
4. S3 Client → AWS S3 버킷 업로드 (또는 로컬 저장소)
5. 데이터베이스 → 파일 메타데이터 저장 (URL, 크기, 형식)
6. 클라이언트 ← 파일 URL 응답
```

## 보안 아키텍처

### 인증 및 인가
1. **JWT 기반 인증**: access token + refresh token 전략
2. **RBAC (역할 기반 접근 제어)**: customer, expert, admin, sub-expert 역할
3. **토큰 갱신**: 자동 토큰 갱신 메커니즘
4. **세션 관리**: Redis 기반 세션 저장 (선택적)

### 데이터 보호
1. **비밀번호 해싱**: bcrypt (솔트 + 다중 라운드)
2. **개인정보 암호화**: 데이터베이스 수준 암호화 (선택적)
3. **감사 로그**: 모든 중요한 작업 로깅 (audit_logs)
4. **데이터 마스킹**: 민감한 정보 응답 시 마스킹

### API 보안
1. **HTTPS 강제**: 모든 통신 TLS 암호화
2. **CORS 제한**: 허용된 오리진만 접근 가능
3. **Rate Limiting**: IP/사용자별 API 요청 제한
4. **SQL 인젝션 방지**: Prisma 파라미터화된 쿼리
5. **XSS 방지**: Helmet.js 헤더, 입력값 검증
6. **CSRF 보호**: SameSite 쿠키, CSRF 토큰

### 인프라 보안
1. **방화벽**: 필요한 포트만 개방
2. **보안 그룹**: AWS/GCP 보안 그룹 구성
3. **정기 업데이트**: OS, 패키지 정기 업데이트
4. **모니터링**: 의심스러운 활동 감지 및 알림

## 기술 결정 사항

### 백엔드 프레임워크: Fastify (Express 대신)
**선택 이유**:
- 높은 성능 (낮은 오버헤드, 빠른 요청 처리)
- 플러그인 기반 아키텍처 (모듈화 용이)
- 내장 유효성 검증 (TypeBox 통합)
- 더 나은 TypeScript 지원
- 활발한 생태계 및 커뮤니티

### ORM: Prisma
**선택 이유**:
- 타입 안전성 (자동 생성 TypeScript 타입)
- 직관적인 쿼리 API
- 마이그레이션 관리 용이
- 성능 최적화 (연결 풀링, 쿼리 최적화)
- 관계형 데이터 모델링 강점

### UI 프레임워크: Ant Design (Tailwind 대신)
**선택 이유**:
- 전문가 웹앱에 적합한 모바일 컴포넌트
- 백오피스에 적합한 풍부한 관리자 컴포넌트
- 일관된 디자인 시스템
- 테마 커스터마이징 가능성
- 한국어 문서 및 커뮤니티 지원

### 상태 관리: Zustand (Redux 대신)
**선택 이유**:
- 간결한 API (보일러플레이트 최소화)
- 타입 안전성 (TypeScript 친화적)
- 미들웨어 지원 (persist, devtools)
- React 18 동시성 기능 호환
- 학습 곡선 낮음

### 실시간 통신: Socket.io
**선택 이유**:
- 폴백 메커니즘 (WebSocket 불가 시 HTTP 롱폴링)
- 방 기능 (채팅, 알림 그룹화)
- 이벤트 기반 아키텍처
- 확장성 (Redis 어댑터)
- 안정적인 연결 관리
