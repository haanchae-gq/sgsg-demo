## 서비스 관리

### 1. 서비스 카테고리 관리 (3-tier 계층 구조) - 시나리오 1
**UI 구성**: Ant Design Tree + Modal + Form

#### 상세 작업 흐름

1. 관리자 로그인  
2. 백오피스 → 메뉴 03 서비스관리 진입  
3. 서비스 카테고리 트리 구조 확인  
   - 기존 대·중·소분류 목록 확인  
   - 신규 추가가 필요한 서비스 영역 판단  
4. 대분류 생성  
   - “대분류 생성” 클릭  
   - 대분류명 입력  
   - 사용 여부(is_active) 설정  
   - 저장  
5. 시스템 처리  
   - 2자리 숫자 코드 자동 생성(예: 04)  
   - 코드 수정 불가 처리  
   - 생성 이력 로그 저장  
   - 하위 중분류 생성 가능 상태로 반영  
6. 생성된 대분류 선택  
7. 중분류 생성  
   - “중분류 생성” 클릭  
   - 중분류명 입력  
   - 사용 여부(is_active) 설정  
   - 멤버십 적용 가능 여부(membership_available) 설정  
   - 저장  
8. 시스템 처리  
   - 코드 자동 생성(예: 0401)  
   - membership_available 기본값 false  
   - 변경 이력 로그 저장  
   - 배정 로직 관리 메뉴에서 참조 가능 상태로 등록  
   - 전문가관리 메뉴에서 멤버십 가입 가능 대상 중분류로 노출 여부 결정  
9. 멤버십 적용 여부 설정 판단  
   - 해당 서비스가 멤버십 우선 배정 대상인지 운영 정책 검토  
   - membership_available=true 설정 시 영향 범위 안내 팝업 표시  
     · 현재 활동 전문가 수  
     · 해당 서비스 예상 수요  
10. 소분류 생성  
    - 생성된 중분류 선택  
    - “소분류 생성” 클릭  
    - 소분류명 입력  
    - 사용 여부(is_active) 설정  
    - 저장  
11. 시스템 처리  
    - 코드 자동 생성(예: 040101)  
    - 주문 생성 시 필수 선택 단위로 등록  
    - 기본비용·현장비용·수수료 설정 가능 상태로 전환  
    - 코드 값은 수정 불가 처리  
12. 기본비용 설정 필요 알림 표시  
    - 소분류 생성 직후 “기본비용 미설정” 상태 표시  
    - 기본비용 미설정 시 주문 생성 불가 안내  
13. 배정 로직 연동 확인  
    - membership_available=true 인 중분류일 경우  
    - 배정 로직 관리 메뉴에서 해당 중분류가 멤버십 필터 기준으로 반영됨  
    - 멤버십 전문가가 존재할 경우 자동배정 후 HOLD 정책 적용 구조 확인  
14. 주문관리 연동 확인  
    - 주문 생성 화면에서 신규 소분류 선택 가능 여부 확인  
    - 주문 생성 시 다음 항목 스냅샷 저장 확인  
      · 대분류 코드  
      · 중분류 코드  
      · 소분류 코드  
      · membership 적용 기준 중분류 코드  
15. 전문가관리 연동 확인  
    - 전문가관리 → 서비스 가능 범위 설정 화면 진입  
    - 신규 소분류가 선택 가능 목록에 노출되는지 확인  
    - membership_available=true 인 경우  
      · 해당 중분류에 대해 멤버십 가입 가능 여부 확인  
16. 전문가웹앱 연동 확인  
    - 전문가 프로필에서 수행 가능 서비스 설정 시 신규 소분류 노출 확인  
    - 멤버십 대상 서비스일 경우  
      · 멤버십 가입 서비스 목록에 해당 중분류 노출 확인  
17. 카테고리 비활성화 테스트  
    - 소분류 is_active=false 변경  
    - 주문 생성 화면에서 선택 불가 처리 확인  
    - 기존 주문에는 영향 없음 확인  
18. 변경 이력 검증  
    - 생성·수정·비활성화 이력 로그 조회  
    - 변경 전/후 값, 변경 주체, 변경 시각 기록 확인  
19. 예외 처리  
    - 필수값 누락 시 저장 불가  
    - 코드 중복 생성 불가  
    - 트랜잭션 단위 처리로 부분 생성 방지  
20. 최종 상태  
    - 서비스 카테고리 구조 확정  
    - 중분류 단위 멤버십 적용 기준 확정  
    - 소분류 단위 주문 생성 기준 확정  
    - 배정·정산·전문가웹앱에서 참조 가능한 상태로 반영 완료  

#### 대분류 관리 (Level 1)
- **생성**: "대분류 생성" 버튼 → 이름, 설명, 활성화 여부 입력 → 2자리 숫자 코드 자동 생성
- **수정/삭제**: 트리 노드 우클릭 또는 편집 버튼
- **비즈니스 규칙**: 
  - 코드 자동 생성 (예: "04") 후 수정 불가
  - `is_active=false` 시 하위 중분류/소분류 모두 비활성화 상태로 표시
  - 변경 이력 로그 저장 (`audit_logs`)

#### 중분류 관리 (Level 2)
- **생성**: 대분류 선택 → "중분류 생성" → 이름, 설명, 활성화 여부, 멤버십 적용 가능 여부 설정
- **멤버십 적용 가능 여부 (`membership_available`)**:
  - `true` 설정 시 멤버십 배정 로직에서 우선 배정 대상
  - 전문가관리 메뉴에서 멤버십 가입 가능 대상으로 노출
  - 변경 시 영향 범위 안내 팝업 (활동 전문가 수, 예상 수요)
- **비즈니스 규칙**:
  - 코드 자동 생성 (상위 대분류 코드 + 2자리 숫자, 예: "0401")
  - `membership_available` 기본값 `false`

#### 소분류 관리 (Level 3)
- **생성**: 중분류 선택 → "소분류 생성" → 이름, 설명, 활성화 여부 설정
- **비즈니스 규칙**:
  - 코드 자동 생성 (상위 중분류 코드 + 2자리 숫자, 예: "040101")
  - 주문 생성 시 필수 선택 단위
  - 기본비용, 현장비용, 수수료 설정 가능한 최소 단위
- **연동 확인**:
  - 주문관리: 신규 소분류 선택 가능 여부
  - 전문가관리: 서비스 가능 범위 설정에 노출
  - 전문가웹앱: 전문가 프로필 서비스 설정에 노출

### 2. 기본비용 관리 - 시나리오 2
**UI 구성**: 소분류 상세 화면 → "기본비용 설정" 탭

#### 상세 작업 흐름

1. 관리자 로그인  
2. 백오피스 → 메뉴 03 서비스관리 진입  
3. 대·중·소분류 트리에서 기본비용을 설정할 소분류 선택  
4. 소분류 상세 화면 진입  
5. “기본비용 설정” 탭 클릭  
6. 현재 기본비용 설정 상태 확인  
   - base_price 값 존재 여부  
   - 단위(unit_type) 설정 여부  
   - 최소 금액(min_price) 설정 여부  
   - VAT 포함 여부  
   - 적용 시작일(effective_start_date) 존재 여부  
7. 신규 기본비용 등록 또는 수정 결정  
8. 기본비용 입력  
   - base_price 입력  
   - unit_type 선택(EA, 면적, 시간 등)  
   - min_price 입력  
   - vat_included_flag 설정  
   - effective_start_date 설정  
9. 스코프 설정  
   - 적용 범위 선택  
     · 신규 주문에만 적용  
     · 특정 채널에만 적용 여부  
   - 기존 주문에는 영향 없음 원칙 확인  
10. 저장 버튼 클릭  
11. 시스템 사전 검증  
    - base_price ≥ 0 검증  
    - min_price ≤ base_price 여부 검증  
    - effective_start_date가 과거 정산 완료 주문에 영향을 주지 않는지 검증  
    - 동일 소분류에 동일 시작일 중복 정책 존재 여부 검증  
12. 저장 처리  
    - service_category_price 테이블에 신규 row 생성  
    - 이전 가격은 종료일 자동 설정  
    - price_version 증가  
    - 변경 이력 로그 저장  
13. 주문관리 연동 확인  
    - 신규 주문 생성 시 해당 소분류 선택  
    - 기본 금액 자동 세팅 확인  
    - 금액 수정 없이 생성 시 base_price 반영 확인  
14. 결제 로직 연동 확인  
    - WAIT_PAYMENT 상태에서 기본 금액 기준 결제 링크 생성 확인  
    - 금액 수정 시 기존 base_price는 스냅샷 유지 확인  
15. 엑셀 업로드 연동 확인  
    - 해당 소분류 엑셀 템플릿 다운로드  
    - 기본 금액 자동 입력 여부 정책 확인  
    - 업로드 시 base_price 기준 금액 자동 계산 또는 수기 입력 허용 여부 확인  
16. 정산관리 연동 확인  
    - 해당 소분류 기준 수수료 계산이 base_price 기준으로 동작하는지 확인  
    - VAT 포함 여부가 정산 계산식에 반영되는지 확인  
17. 전문가웹앱 연동 확인  
    - 전문가 주문 상세 화면에서 기본 금액 표시 여부 확인  
    - 현장 추가비용이 base_price와 별도로 처리되는지 확인  
18. 기존 주문 영향 검증  
    - 과거 주문의 total_price가 변경되지 않는지 확인  
    - 기존 주문은 주문 스냅샷 기준 금액 유지 확인  
19. 예외 케이스 처리  
    - 기본비용 미설정 상태에서 소분류 활성화 시 경고 표시  
    - base_price 0원 설정 시 무료 서비스 여부 정책 확인  
    - effective_start_date 미래 설정 시 해당 일자 이후 주문에만 적용 확인  
20. 최종 상태  
    - 소분류 기준 기본비용 정책 확정  
    - 신규 주문은 최신 price_version 기준 적용  
    - 기존 주문은 생성 시점 스냅샷 유지  
    - 정산·배정·전문가웹앱과 정합성 유지  

#### 기본비용 등록/수정
- **입력 항목**: `base_price`, `unit_type` (EA/면적/시간 등), `min_price`, `vat_included`, `effective_start_date`
- **스코프 설정**: 신규 주문에만 적용, 특정 채널에만 적용 여부
- **시스템 검증**:
  - `base_price ≥ 0`, `min_price ≤ base_price`
  - 동일 소분류에 동일 `effective_start_date` 중복 불가
  - 과거 정산 완료 주문 영향 검증
- **버전 관리**:
  - 신규 가격 생성 시 기존 가격 `effective_end_date` 자동 설정
  - `price_version` 증가
  - 변경 이력 로그 저장

#### 연동 확인
- **주문관리**: 신규 주문 생성 시 기본 금액 자동 세팅
- **결제 로직**: `WAIT_PAYMENT` 상태에서 기본 금액 기준 결제 링크 생성
- **정산관리**: `base_price` 기준 수수료 계산, VAT 포함 여부 반영
- **엑셀 업로드**: 템플릿 다운로드 시 기본 금액 자동 입력 여부 정책

### 3. 현장비용 카테고리 관리 - 시나리오 3
**UI 구성**: "현장비용 관리" 탭 → 현장비용 카테고리 목록 + 생성/수정

#### 상세 작업 흐름

1. 관리자 로그인  
2. 백오피스 → 메뉴 03 서비스관리 진입  
3. 대·중·소분류 트리에서 현장비용을 적용할 소분류 선택  
4. 소분류 상세 화면 진입  
5. “현장비용 관리” 탭 클릭  
6. 기존 현장비용 항목 목록 확인  
   - 현장비용명  
   - 금액 또는 단가  
   - 과금 단위  
   - 사용 여부(is_active)  
   - 정산 반영 여부  
7. 신규 현장비용 카테고리 생성 클릭  
8. 현장비용 기본 정보 입력  
   - 현장비용명 입력  
   - 비용 설명 입력  
   - 과금 방식 선택  
     · 고정금액(FIXED)  
     · 단가×수량(UNIT_BASED)  
   - 기본 금액 입력  
   - VAT 포함 여부 설정  
   - 정산 반영 여부 설정  
   - 사용 여부(is_active) 설정  
9. 스코프 설정  
   - 해당 소분류에만 적용  
   - 특정 중분류 전체 적용 여부 선택 가능  
   - 채널별 적용 여부 설정 가능  
10. 저장 버튼 클릭  
11. 시스템 사전 검증  
    - 금액 ≥ 0 검증  
    - 동일 소분류 내 동일 명칭 중복 검증  
    - 과금 방식에 따른 필수값 검증  
    - 정산 반영 여부 true일 경우 수수료 계산 로직 충돌 여부 확인  
12. 저장 처리  
    - service_on_site_fee 테이블에 row 생성  
    - 소분류와 매핑 테이블 생성  
    - created_at, created_by 기록  
    - 변경 이력 로그 저장  
13. 주문관리 연동 확인  
    - 해당 소분류 주문 생성  
    - 주문 상세 화면에서 “현장비용 추가” 가능 여부 확인  
    - 운영자가 금액 수정 시 현장비용 항목 선택 가능 여부 확인  
14. 전문가웹앱 연동 확인  
    - 전문가 주문 상세 화면 진입  
    - 서비스 진행 중 현장 추가금 입력 가능 여부 확인  
    - 등록된 현장비용 카테고리가 드롭다운에 노출되는지 확인  
    - 수량 기반 항목일 경우 수량 입력 필드 노출 확인  
15. 금액 계산 구조 확인  
    - 최종 total_price 계산식  
      · base_price  
      - 현장비용 합계  
      - 할인 금액  
    - paid_amount_total 대비 remaining_payable_amount 재계산 확인  
16. 결제 연동 확인  
    - 서비스 중 현장비용 추가 발생  
    - remaining_payable_amount 증가 확인  
    - 결제 링크 재발송 시 증가 금액 반영 확인  
17. 환불 연동 확인  
    - 현장비용 포함 결제 완료 상태에서 일부 환불 처리  
    - 환불 금액 계산 시 현장비용 포함 여부 확인  
18. 정산관리 연동 확인  
    - 정산 반영 여부 true일 경우  
      · 전문가 정산 금액에 포함되는지 확인  
      · 채널 수수료 계산식 반영 여부 확인  
    - 정산 반영 여부 false일 경우  
      · 플랫폼 수익으로만 처리되는지 확인  
19. 엑셀 업로드 연동 확인  
    - 해당 소분류 엑셀 템플릿에 현장비용 항목이 포함되는지 여부 확인  
    - 업로드 시 현장비용 컬럼 자동 인식 여부 확인  
20. 기존 주문 영향 검증  
    - 신규 현장비용 생성 전 주문에는 자동 반영되지 않음 확인  
    - 기존 주문에 수동 추가만 가능하도록 제한  
21. 비활성화 테스트  
    - is_active=false 설정  
    - 신규 주문 및 전문가웹앱에서 선택 불가 확인  
    - 기존 주문 데이터 유지 확인  
22. 예외 처리  
    - 현장비용 금액 수정 시 가격 이력 로그 관리  
    - 동일 명칭으로 다른 금액 생성 불가  
    - 단가 기반 항목에서 수량 0 또는 협의 허용  
23. 최종 상태  
    - 소분류 기준 현장비용 카테고리 생성 완료  
    - 주문관리·전문가웹앱·정산관리와 정합성 유지  
    - 금액 계산 구조 일관성 확보

#### 현장비용 카테고리 생성
- **입력 항목**: 현장비용명, 설명, `fee_type` (FIXED/UNIT_BASED), `base_amount`, `vat_included`, `settlement_included`, `is_active`
- **스코프 설정**: 해당 소분류에만 적용, 특정 중분류 전체 적용, 채널별 적용 여부
- **시스템 검증**:
  - 금액 ≥ 0 검증
  - 동일 소분류 내 동일 명칭 중복 검증
  - 과금 방식에 따른 필수값 검증
  - `settlement_included=true`일 경우 수수료 계산 로직 충돌 여부 확인

#### 연동 확인
- **주문관리**: 주문 상세 화면에서 "현장비용 추가" 가능 여부
- **전문가웹앱**: 서비스 진행 중 현장 추가금 입력 가능, 등록된 현장비용 카테고리 드롭다운 노출
- **금액 계산**: `total_price = base_price + 현장비용 합계 - 할인 금액`
- **결제 연동**: 서비스 중 현장비용 추가 발생 시 `remaining_payable_amount` 증가, 결제 링크 재발송
- **정산관리**: `settlement_included` 여부에 따라 전문가 정산 금액 포함/제외

   - 기본 금액  
   - 사용 여부(is_active)

5. 신규 현장비용 카테고리 생성 클릭

16. 저장 버튼 클릭

18. 우선순위 규칙 적용

19. 주문관리 연동 확인  
    - 해당 소분류 주문 생성  
    - 주문 상세에서 “현장비용 추가” 클릭

20. 전문가웹앱 연동 확인  
    - 서비스 진행 중 현장비용 추가 화면 진입  
    - 매핑된 항목만 드롭다운에 표시되는지 확인  
    - 수량 기반 항목은 수량 입력 필드 표시 확인  
21. 금액 계산 연동 확인  
    - base_price + 선택된 현장비용 합계 = total_price 반영 확인

22. 정산관리 연동 확인

23. 엑셀 업로드 연동 확인  
    - 해당 소분류 템플릿 다운로드

26. 예외 처리

    - 중복 매핑 저장 불가  
27. 최종 상태  
    - 현장비용 카테고리 마스터 생성 완료  
    - 서비스 카테고리 기준 매핑 완료  
    - 주문·전문가웹앱·정산과 정합성 유지  
    - 매핑 레벨 우선순위 규칙 확정  

##### 시나리오 5: 서비스–현장비용 매핑 수정 및 우선순위 변경 운영

1. 관리자 로그인  
2. 백오피스 → 메뉴 03 서비스관리 진입  
3. “서비스–현장비용 매핑 관리” 탭 선택  
4. 매핑 기준 선택  
   - 중분류 기준 매핑 조회  
   - 소분류 기준 매핑 조회  
5. 특정 서비스 카테고리 선택  
6. 현재 매핑 상태 확인  
   - 매핑된 현장비용 목록  
   - 매핑 레벨(LEVEL2 또는 LEVEL3)  
   - 필수 여부  
   - 최대 수량 제한  
   - 적용 시작일  
7. 수정 필요 사유 확인  
   - 특정 현장비용 항목 제거 필요  
   - 신규 현장비용 추가 필요  
   - 필수 여부 변경 필요  
   - 중분류 매핑을 소분류 예외 매핑으로 전환 필요  
8. 매핑 수정 액션 실행  
9. 현장비용 항목 추가  
   - 추가할 현장비용 선택  
   - 필수 여부 설정  
   - 수량 제한 설정  
   - 적용 시작일 설정  
10. 현장비용 항목 제거  
    - 기존 매핑 항목 선택  
    - 제거 클릭  
    - 제거 사유 입력  
11. 우선순위 변경 처리  
    - 중분류 매핑이 존재하는 상태에서  
      · 특정 소분류에 별도 매핑 추가 시  
      · 해당 소분류는 중분류 매핑을 덮어씀  
12. 시스템 우선순위 규칙 적용  
    - 소분류 매핑 존재 시 → 해당 매핑 우선 적용  
    - 소분류 매핑 미존재 시 → 중분류 매핑 적용  
13. 저장 버튼 클릭  
14. 시스템 사전 검증  
    - 중복 매핑 여부 확인  
    - 필수 항목 제거 시 정책 위반 여부 확인  
    - 적용 시작일이 과거 주문에 영향 주지 않는지 확인  
15. 저장 처리  
    - 기존 매핑 row 종료일 자동 설정  
    - 신규 매핑 row 생성  
    - mapping_version 증가  
    - 변경 이력 로그 저장  
16. 기존 주문 영향 검증  
    - 이미 생성된 주문에는 기존 매핑 스냅샷 유지  
    - 신규 주문부터 변경된 매핑 적용  
17. 주문관리 연동 확인  
    - 해당 소분류 신규 주문 생성  
    - 현장비용 추가 화면에서 변경된 매핑 반영 확인  
18. 전문가웹앱 연동 확인  
    - 서비스 진행 중 현장비용 입력 화면 진입  
    - 매핑 변경 내용 즉시 반영 확인  
19. 정산 연동 확인  
    - 정산 반영 여부가 변경된 경우  
    - 신규 주문 정산 계산식 반영 확인  
20. 엑셀 템플릿 연동 확인  
    - 해당 소분류 엑셀 템플릿 다운로드  
    - 매핑된 현장비용 컬럼 업데이트 여부 확인  
21. 중분류 매핑 → 소분류 예외 전환 시나리오  
    - 기존 중분류 매핑 존재  
    - 특정 소분류만 예외 처리 필요  
    - 해당 소분류에 별도 매핑 생성  
    - 시스템은 자동으로 해당 소분류에 중분류 매핑 적용 제외  
22. 소분류 매핑 제거 시 처리  
    - 소분류 매핑 삭제  
    - 자동으로 중분류 매핑 적용 복원  
23. 비활성화 테스트  
    - 현장비용 카테고리 is_active=false 처리  
    - 매핑은 유지하되 신규 주문에서는 선택 불가 확인  
24. 감사 로그 확인  
    - 변경 전 매핑 목록  
    - 변경 후 매핑 목록  
    - 변경 사유  
    - 변경 관리자  
    - 변경 시각  
25. 예외 처리  
    - 매핑 수정 중 주문 동시 생성 충돌 방지 위해 트랜잭션 처리  
    - 매핑 없는 상태에서 필수 입력 요구하지 않도록 검증  
    - 정산 확정 주문에는 영향 없음  
26. 최종 상태  
    - 서비스–현장비용 매핑 정책 최신화  
    - 우선순위 규칙 명확화  
    - 신규 주문에만 적용  
    - 기존 주문 데이터 보존  

#### 매핑 생성/수정
- **매핑 기준**: 중분류 기준 매핑 (LEVEL2) 또는 소분류 기준 매핑 (LEVEL3)
- **매핑 항목**: 현장비용 카테고리 선택, 필수 여부(`is_required`), 최대 수량 제한(`max_quantity`)
- **우선순위 규칙**:
  - 소분류 매핑 > 중분류 매핑 (동일 중분류 하위 소분류에 별도 매핑 존재 시 소분류 매핑 우선 적용)
  - 소분류 매핑 없을 경우 중분류 매핑 적용
- **버전 관리**:
  - 신규 매핑 생성 시 기존 매핑 `effective_end_date` 자동 설정
  - `mapping_version` 증가
  - 변경 이력 로그 저장

#### 매핑 변경 운영
- **항목 추가/제거**: 특정 현장비용 항목 제거, 신규 항목 추가, 필수 여부 변경
- **중분류 → 소분류 예외 전환**: 기존 중분류 매핑 존재 시 특정 소분류에 별도 매핑 생성으로 예외 처리
- **연동 확인**:
  - 주문관리: 신규 주문에서 매핑된 현장비용 항목만 노출
  - 전문가웹앱: 매핑 변경 내용 즉시 반영
  - 엑셀 템플릿: 매핑된 현장비용 컬럼 업데이트 여부 확인

### 5. 채널별 수수료 관리 - 시나리오 6
**UI 구성**: 소분류 상세 화면 → "채널별 수수료 관리" 탭

#### 상세 작업 흐름

6. 현재 수수료 설정 현황 확인  
   - channel_code  
   - channel_name  
   - commission_type  
   - commission_value  
   - 적용 시작일(effective_start_date)

7. 신규 채널 수수료 설정 클릭  
8. 수수료 정보 입력  
   - channel_code 선택  
   - commission_type 선택  
     · RATE: 퍼센트 수수료

   - 적용 시작일 설정

    - commission_value ≥ 0 검증  
    - RATE일 경우 0 ≤ value ≤ 100 검증

12. 주문관리 연동 확인

    - 전문가 정산 금액 계산식 확인  
      · total_price 기준 계산  
      · 현장비용 포함 여부 확인  
      · 수수료 적용 방식 확인  
14. 금액 수정 연동 확인  
    - 주문 금액 수정 시  
    - 수수료 계산이 최신 price_snapshot 기준으로 재계산되는지 확인  
15. 환불 연동 확인  
    - 부분 환불 발생 시  
    - 환불 금액에 대한 수수료 재계산 정책 확인  
    - 수수료 반환 여부 정책 적용 확인  
16. 엑셀 업로드 연동 확인  
    - 엑셀 업로드 시 channel_code 입력  
    - 업로드된 주문에 해당 수수료 자동 적용 확인  
17. 채널별 예외 정책 설정  
    - 특정 채널은 수수료 0원 설정 가능  
    - 특정 채널은 FIXED + RATE 혼합 구조 불가  
18. 수수료 수정 시나리오  
    - 기존 채널 수수료 값 변경  
    - 적용 시작일을 미래로 설정

    - 이후 신규 주문에 적용되지 않음 확인

    - 변경 후 수수료 값  
    - 변경 사유  
    - 변경 관리자  
    - 변경 시각  
21. 예외 처리  
    - 수수료 미설정 채널 주문 생성 시 경고 표시  
    - channel_code 없는 주문 생성 불가 처리  
    - 정산 확정 주문에는 수수료 변경 소급 적용 불가  
22. 최종 상태  
    - 소분류 기준 채널별 수수료 정책 확정  
    - 신규 주문에만 적용  
    - 기존 주문 스냅샷 유지  
    - 정산관리와 정합성 유지  

#### 수수료 설정
- **입력 항목**: `channel_code`, `commission_type` (RATE/FIXED), `commission_value`, `vat_basis`, `effective_start_date`
- **시스템 검증**:
  - 동일 소분류 + 동일 `channel_code` + 동일 `effective_start_date` 중복 불가
  - `commission_value ≥ 0`, RATE일 경우 0 ≤ value ≤ 100
  - 적용 시작일이 과거 확정 정산 주문에 영향 없는지 검증
- **버전 관리**:
  - 신규 수수료 생성 시 기존 수수료 `effective_end_date` 자동 설정
  - `commission_version` 증가
  - 변경 이력 로그 저장

- **주문관리**: 주문 생성 시 `channel_code` 스냅샷 저장, 정산 예정 금액 계산 시 수수료 반영
- **정산관리**: 전문가 정산 금액 계산식에 수수료 적용 방식 확인
- **금액 수정**: 주문 금액 수정 시 수수료 계산이 최신 `price_snapshot` 기준 재계산
- **환불 연동**: 부분 환불 발생 시 환불 금액에 대한 수수료 재계산 정책 적용

### 6. 추가입력항목 관리 - 시나리오 7
**UI 구성**: 서비스 카테고리 상세 화면 → "추가입력항목" 탭
#### 상세 작업 흐름

1. 관리자 로그인
2. 백오피스 → 메뉴 03 서비스관리 진입
3. 추가입력항목을 설정할 서비스 카테고리 선택
  - 대·중·소분류 트리 확인
  - 기준 정책
    · 중분류 기준으로 공통 추가입력항목을 관리할지
    · 특정 소분류에만 예외 항목을 둘지 결정
  - 운영자는
    · “중분류 기준으로 공통 필드 설정 → 일부 소분류에서 예외 필드 추가/제거” 패턴을 기본 전략으로 사용
4. 추가입력항목 탭 진입
  - 선택한 카테고리(중분류 또는 소분류)의 상세 화면으로 이동
  - 상단 탭에서 “추가입력항목” 탭 클릭
  - 현재 정의된 항목 목록 확인
    · field_key
    · 표시명(label)
    · 필드 타입(field_type: TEXT/NUMBER/DATE/SELECT 등)
    · 필수 여부(required)
    · 노출 위치(주문관리/전문가웹앱/양쪽)
    · 사용 여부(is_active)
    · 정렬 순서(sort_order)
5. 신규 추가입력항목 생성
  - “필드 추가” 버튼 클릭
  - 필수 정보 입력
    · field_key 입력
      - 영문/숫자 조합, 고유값
      - 한번 생성되면 수정 불가
    · 표시명(label) 입력
      - 운영/CS/전문가 모두 이해 가능한 용어 사용
    · 필드 타입 선택
      - TEXT: 자유 입력
      - NUMBER: 숫자 입력
      - DATE: 날짜 선택
      - SELECT: 사전 정의 옵션 중 선택
    · 필수 여부 설정(required_flag)
      - 주문 생성/수정 시 반드시 입력되어야 하는지
    · 노출 위치 설정
      - ORDER_BACKOFFICE_ONLY
      - EXPERT_APP_ONLY
      - BOTH
    · 정렬 순서(sort_order) 지정
    · SELECT 타입일 경우 옵션 목록 입력
      - option_key / option_label 리스트
  - 저장 클릭

6. 시스템 사전 검증
  - field_key 중복 여부 확인
  - 필드 타입에 따른 필수 속성 검증
    · NUMBER 타입: 최소/최대값 설정 시 min ≤ max
    · SELECT 타입: 최소 1개 이상의 옵션 필수
  - 동일 카테고리 내 동일 label을 허용할지 정책에 따라 경고 또는 허용
  - 저장 가능하면 DB에 신규 필드 정의 생성

7. 저장 처리 및 다른 메뉴 연동 준비
  - service_category_extra_field 테이블에 row 생성
    · category_id (중분류/소분류 ID)
    · field_key, field_type, label, options, required_flag, is_active, sort_order
  - 엑셀 템플릿 정의에 해당 필드 포함 플래그 설정
  - 전문가웹/앱 주문 상세 화면 스키마에 즉시 반영

8. 추가입력항목 변경(수정)
  - 기존 필드 행 선택 후 “수정” 클릭
  - 수정 가능 항목
    · label (표시명)
    · 설명/헬프 텍스트
    · required_flag (필수 여부) – 단, 이미 대량 사용 중이면 경고
    · 노출 위치
    · sort_order
    · SELECT 옵션 label (option_key는 가능하면 불변)
  - 수정 불가 항목
    · field_key
    · field_type (TEXT → NUMBER 등 타입 변경은 별도 신규 필드 생성 권장)
  - 저장 클릭

9. 필드 수정의 영향도 안내
  - required_flag를 true로 변경하는 경우
    · 기존 주문 중 해당 값이 비어 있는 주문 수를 조회하여 안내
  - 노출 위치를 EXPERT_APP_ONLY → BOTH로 변경 시
    · 주문 생성 화면에서도 입력 필드가 생김을 안내
  - SELECT 옵션을 제거/변경하는 경우
    · 기존 주문에 저장된 option_key는 그대로 유지되며,
  화면에서는 “(이전 옵션)” 등으로 표시 필요

10. 추가입력항목 비활성화
  - 더 이상 사용하지 않을 필드의 “사용 여부”를 OFF로 변경
  - 시스템 처리
    · 신규 주문 및 수정 화면에서 입력 불가
    · 전문가웹/앱에서도 입력/수정 불가
    · 기존 주문 상세에서는 “읽기 전용”으로 값만 표시
  - 필드 삭제는 원칙적으로 허용하지 않음
    · 데이터 손실 방지
    · 필요 시 “완전 숨김” 플래그 추가 고려(슈퍼관리자 전용)
11. 중분류 vs 소분류 기준 상속/우선순위
  - 중분류에 정의된 추가입력항목은
    · 기본적으로 해당 중분류 하위 모든 소분류에 공통으로 적용
  - 소분류에 별도 추가입력항목을 정의한 경우
    · 해당 소분류에는 중분류 공통 필드 + 소분류 전용 필드가 모두 노출
  - 소분류에서 “중분류 공통 필드를 비활성화”하는 예외 정책이 필요하다면
    · 소분류 수준에서 override_flag를 설정해
  특정 중분류 필드를 숨김 처리할 수 있도록 확장 가능
  - 우선순위 규칙
    · “보이는 필드 = (중분류 공통 필드 – 소분류 숨김 지정) + 소분류 전용 필드”
12. 메뉴 02 주문관리와의 연동 확인
  - 해당 카테고리 소분류로 신규 주문 생성
  - 주문 생성 화면에서 추가입력항목이 노출되는지 확인
  - 필수 필드 누락 시 저장 불가 여부 확인
  - 주문 상세 화면에서도 동일 필드들이 조회되는지 확인
  - 주문 수정 시에도 동일 검증이 작동하는지 확인


13. 전문가웹/앱과의 연동 확인
  - 전문가에게 배정된 동일 카테고리 주문 확인
  - 전문가웹/앱 주문 상세 화면에서 추가입력항목 노출 확인
  - 노출 위치가 EXPERT_APP_ONLY 필드만 별도로 표시되는지 확인
  - 전문가가 현장에서 값을 수정/추가 가능한 필드와
  백오피스에서만 수정 가능한 필드의 구분이 올바른지 검증


14. 엑셀 템플릿 연동 확인
  - 메뉴 02 → 엑셀 템플릿 다운로드(해당 카테고리 기준)
  - 추가입력항목 필드가 컬럼으로 포함되는지 확인
  - 필드 비활성화 시 템플릿에서 컬럼 유지 여부 정책 결정
    · 컬럼은 남기되 “더 이상 사용하지 않음” 안내 주석 추가
    · 신규 템플릿 버전부터는 컬럼 자체 제거 등 정책 선택


15. 데이터 저장 구조 확인
  - 주문 생성 시
    · order_extra_field_value 테이블에
  · order_id
  · field_key
  · field_value
  형태로 저장
  - 중분류/소분류 필드 모두 동일한 구조로 저장
  - 향후 분석/리포트 시 field_key 기준으로 통계 가능


16. 변경 이력 및 감사 로그 확인
  - 추가입력항목 생성/수정/비활성화 이력 조회
  - 로그 항목
    · category_id
    · field_key
    · 변경 전/후 값
    · 변경 관리자
    · 변경 시각
  - 필수 여부(required) 변경은 별도 강조 표시


17. 예외 케이스 처리
  - 현장비용과의 혼동 방지
    · 추가입력항목은 “정보 필드”이고
  현장비용은 “금액 필드”임을 명확히 구분
  - 필드 타입 변경 필요 시
    · 기존 필드를 비활성화하고
    · 새로운 field_key로 재생성하는 것을 원칙으로 한다
  - 사람이 직접 문자열로 적는 대신
  SELECT 옵션으로 최대한 구조화하여
  나중에 통계/정책에 활용 가능하도록 유도


18. 최종 상태
  - 서비스 카테고리별 추가입력항목 정의가 완료된다
  - 주문관리, 전문가웹/앱, 엑셀 템플릿이 모두 이 정의를 단일 진실원으로 참조한다
  - 필드 변경/비활성화가 과거 주문 데이터에 피해를 주지 않도록 설계된다


#### 필드 생성/수정
- **입력 항목**: `field_key` (영문/숫자 조합, 고유, 수정 불가), `label`, `field_type` (TEXT/NUMBER/DATE/SELECT 등), `options` (SELECT 타입), `is_required`, `display_location` (ORDER_BACKOFFICE_ONLY/EXPERT_APP_ONLY/BOTH), `sort_order`
- **상속/우선순위 규칙**:
  - 중분류 필드는 하위 소분류에 공통 적용
  - 소분류 전용 필드는 중분류 필드와 병합 노출
  - 소분류에서 중분류 공통 필드 비활성화 가능 (`override_flag`)
- **비활성화 처리**: `is_active=false` 설정 시 신규 주문 및 수정 화면에서 입력 불가, 기존 주문은 읽기 전용 표시

#### 연동 확인
- **주문관리**: 주문 생성/수정 화면에서 추가입력항목 노출, 필수 필드 검증
- **전문가웹앱**: `display_location`에 따른 필드 노출 구분 검증
- **엑셀 템플릿**: 추가입력항목 필드 컬럼 포함 여부 확인
- **데이터 저장**: `order_extra_field_value` 테이블에 `field_key`/`field_value` 형태 저장

### 7. 이미지 관리 (AWS S3 통합)
- **서비스 이미지 업로드**: Ant Design Upload 컴포넌트로 S3 직접 업로드
- **이미지 리사이징**: Lambda@Edge로 다양한 크기 자동 생성 (썸네일, 상세)
- **CDN 캐싱**: CloudFront로 이미지 빠른 전송

### 8. 서비스 검색 및 필터링
- **실시간 검색**: 대분류/중분류/소분류 이름으로 검색
- **활성화 필터**: 활성/비활성 서비스 필터링
- **가격 범위 필터**: 최소/최대 가격으로 서비스 필터링
- **멤버십 가능 필터**: `membership_available=true` 인 중분류만 표시
